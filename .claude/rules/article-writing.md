# 記事執筆ルール

## 見出しはストーリーを描く

見出し（`##` / `###`）は、読者を次へ引っ張る「物語の一歩」として書く。見出しだけを上から拾い読みしても、記事の流れ（問い → 仕組み → データ → 答え）が伝わるように並べる。

**必ず守る:**

- **見出しに読点「、」を入れない**
- 本文の一文をそのまま切り出したような、だらだら長い説明調にしない（特に読点でつなぐ形）

**推奨:**

- 端的な名詞句・体言止め、または短い問いかけ（「そもそも〜とは何か」など）
- **導入の見出しは、いきなり専門用語で始めず読者の疑問から入る**。急に技術用語が現れると「今までの話は何だったのか」という断絶が生まれる
- 専門用語を出す場合も、その語が何をするものかを添える（例：「〜という指標」より「速度の損失だけを取り出す『機会損失率』」）

**書き換え例:**

| ✗ 読点あり・唐突・説明調 | ○ 流れを描く |
|---|---|
| サイトスピードは「対数正規分布」を描く（唐突に専門用語） | そもそも「速くなった」とは何か |
| 期間の代表値は「最頻値」 | 速さを一つの数字で表す「最頻値」 |
| 「機会損失率」という指標 | 速度の損失だけを取り出す「機会損失率」 |
| あるサイトの最頻値は、月々こう動いた（読点・説明調） | 実例で見る最頻値の月次推移 |

本文（見出し以外）では読点を通常どおり使ってよい。あくまで**見出しのルール**。
